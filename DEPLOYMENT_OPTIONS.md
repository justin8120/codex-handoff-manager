# DEPLOYMENT_OPTIONS.md

No deployment target has been selected yet. This project is currently validated locally with:

```bash
npm run validate
```

The production build is generated by Vite under `dist/`.

## Option 1: GitHub Pages

### Fit

GitHub Pages is a good fit for a static Vite app when the repository will live on GitHub and the team wants a simple, low-cost deployment path.

### Pros

- Built into GitHub repositories.
- No separate hosting account required.
- Works well for static assets generated by `npm run build`.
- Pairs naturally with the existing GitHub Actions CI setup.

### Cons

- Requires configuring Vite `base` if deployed under a project subpath.
- Fewer preview/deployment workflow features than Vercel or Netlify.
- Custom domains and cache behavior require extra GitHub Pages configuration.

### Good When

- The app is documentation-like or internal handoff tooling.
- GitHub is already the source of truth.
- Minimal hosting complexity is preferred.

## Option 2: Vercel

### Fit

Vercel is a strong fit for modern frontend apps and gives excellent preview deployments for every branch or pull request.

### Pros

- Very smooth Vite support.
- Preview deployments are excellent for review.
- Simple project import from GitHub.
- Good defaults for static frontend apps.

### Cons

- Requires a Vercel account/project.
- Adds an external platform dependency.
- More deployment capability than this static handoff app currently needs.

### Good When

- Pull request preview URLs are important.
- The project may grow into a more active frontend app.
- The team already uses Vercel.

## Option 3: Netlify

### Fit

Netlify is also a good static-site host for Vite builds, with branch previews and straightforward static asset deployment.

### Pros

- Strong static hosting support.
- Good preview and deploy settings.
- Simple build command and publish directory setup.
- Useful redirects/header configuration if needed later.

### Cons

- Requires a Netlify account/site.
- Adds an external platform dependency.
- Preview workflow is good, but Vercel is often smoother for React/Vite defaults.

### Good When

- The team already uses Netlify.
- Static hosting settings, redirects, or headers may matter later.
- A dedicated static-site platform is preferred.

## Recommendation

The current best fit is **GitHub Pages**.

Reasoning:

- The app is a static Codex handoff manager.
- There is no backend, database, authentication, or server runtime.
- GitHub Actions CI already exists.
- Keeping source control and deployment in GitHub minimizes operational overhead.

Before deploying to GitHub Pages, confirm:

1. The GitHub repository name and whether the site will deploy at the user/org root or a project subpath.
2. Whether Vite needs a `base` value for subpath deployment.
3. Whether screenshot artifacts should remain local-only or be uploaded as CI artifacts in a future workflow.
